All API requests require an API key to be included in the headers.

üóëÔ∏è Delete Job

üîÑ Queue System

The API processes up to 3 videos simultaneously. Additional uploads are queued automatically.

How It Works

  1. Upload video ‚Üí Check available slots
  2. If slots available (< 3 running) ‚Üí Start processing immediately
  3. If all slots busy (3 running) ‚Üí Add to queue
  4. When job completes/terminates ‚Üí Start next queued job
üí° Queue Position: Jobs are processed in FIFO (First In, First Out) order.

üìà Job Lifecycle

Job Flow 
1. QUEUED (if 3 jobs running)
   ‚Üì
2. UPLOADING (file transfer)
   ‚Üì
3. PROCESSING
   ‚îú‚îÄ Splitting frames (20%)
   ‚îú‚îÄ AI Detection (30%)
   ‚îú‚îÄ OCR Processing (50%)
   ‚îî‚îÄ Finalizing results
   ‚Üì
4. COMPLETED

Alternative paths:
- QUEUED ‚Üí TERMINATED
- PROCESSING ‚Üí TERMINATED
- PROCESSING ‚Üí ERROR

‚ú® Best Practices

1. Poll for Updates

Poll the job status endpoint every 2-3 seconds to get real-time updates.

JavaScript Example 
const pollJob = async (jobId) => {
  const interval = setInterval(async () => {
    const res = await fetch(`/api/jobs/${jobId}/status`, {
      headers: { 'X-API-Key': 'your-key' }
    });
    const { job } = await res.json();
    
    if (job.status === 'completed') {
      clearInterval(interval);
      // Download results
    }
  }, 2000);
};

2. Handle Queued Jobs

Check the job status after upload. If status === 'queued', inform the user about the queue position.

3. Clean Up Old Jobs

Delete completed jobs regularly to save disk space:

cURL 
curl -X DELETE http://localhost:3000/api/jobs/JOB_ID \
  -H "X-API-Key: your-key"

4. Video Recommendations

  • Max file size: 500MB
  • Supported formats: MP4, MOV, WebM, MKV, AVI
  • Recommended: 720p or 1080p resolution
  • Frame rate: 30-60 FPS
üí° Performance Tip: Videos are processed at 7 FPS. Every other frame is skipped for efficiency.
